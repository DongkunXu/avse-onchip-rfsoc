## Notes per candidate

- **C0 Reference full-window U-Net (single-config)** ((none) baseline): Static footprint 215% BRAM as 4 bitstreams. Peak-live 2.4MB still ~near budget.
- **C6 Single-engine + global pool (same U-Net)** (Axis 3 (schedule only)): Pools to peak-live 2.4MB. Hand-built offset-addressed engine (path3 validated the datapath). Bounds BRAM to peak-live but that is itself ~near budget -> not enough alone.
- **C5 DDR-staged U-Net (skips to DDR)** (Axis 2 (off-chip staging)): On-chip holds only the active stage; skips live in DDR (double-buffered, latency spare). Lowest quality risk; but keeps full compute + depends on DDR BW. Plan-B/control.
- **C4 Tiled U-Net (temporal tiles + halo)** (Axis 2 (temporal tiling)): 1/8 window tiles. Peak scales with tile length. Reuses reference weights; halo handles conv receptive fields. Strong low-risk fit route.
- **C1 Streaming chunked U-Net (frame-sync)** (Axis 1+2 (streaming)): Process ~75ms chunks with carried receptive-field state. Peak decoupled from the 1.2s window. Pairs with distillation from the 0.37M teacher.
- **C2 Streaming TCN (dilated causal)** (Axis 1 (bounded receptive field)): C=128, 8 dilated layers. State = Σ C*K*dilation (200,320 elems). O(state) not O(sequence). Owner prior art cites LSTM-AVSE; TCN is the conv analogue.
- **C3 STFT-domain mask (frame-sync)** (Axis 1 (representation change)): OUT OF SCOPE per D-2 (time-domain only for now). Kept for record: operate on spectrogram [257 x ~75], predict T-F mask frame-by-frame -> peak ~C*F (32,896 elems). Root-cause fix but needs FPGA STFT/iSTFT. Revisit only if time-domain underdelivers.
- **C7 Conv-TasNet-style time mask (no U-Net skips)** (Axis 1 (remove skip topology)): encoder(stride 16) -> latent[128 x 1200] -> TCN mask -> decoder. NO multi-resolution skips -> NO co-residency wall. Streamed peak ~32,768 elems; even non-streamed ~460,800 (~0.9MB) with no skip wall. Time-domain, frame-synchronous, distillable from the 0.37M teacher. STRONG root-cause candidate.
- **C8 Recompute-skip U-Net (regenerate shallow skips)** (Axis 2 (recompute-vs-store)): Don't store skip0/skip1 (55% of resident); recompute them on demand from a coarse checkpoint. Trades cheap compute for scarce BRAM. ~half the skip residency. Borderline alone; shines COMBINED with tiling/streaming. Reuses the trained weights.
- **C9 Compressed-skip U-Net (learned skip bottleneck)** (Axis 1+4 (capacity knob)): Store shallow skips at reduced channels (32->8, 64->16) -> attacks the dominant C*T directly (~40% less resident). Borderline alone; combine with tiling. Light retrain. Also viable but not separately scored: sub-band multirate (learned filterbank) and dual-path chunked-recurrent (DPRNN-style) — both time-domain, bounded peak.

## Reading the table

- The reference U-Net audio peak-live is **2.4 MB ~= 75% BRAM by itself**; add the shared video+fusion overhead and **C0/C6 land at ~140% — they DO NOT FIT.** This is the sharpened, model-backed form of the validated finding: **time-multiplexing/pooling the same U-Net (Axis 3 alone) cannot fit the system.**
- **Bounding the live temporal extent works**: tiling/streaming (C4/C1) and single-resolution masking (C2/C7) drop peak-live by 6–25x into comfortable-fit territory.
- **The cleanest time-domain attack on the root cause is C7** (Conv-TasNet-style): it has **no multi-resolution U-Net skips at all**, so the skip-residency wall simply does not exist — without leaving the time domain (D-2).
- **Decision context**: D-2 = time-domain only, so **C3 (STFT) is out of scope** (kept for record). C8/C9 are BRAM-reclaim *levers* — best combined with a base architecture, not run standalone.

## Phase-2 set (owner D-8 + analysis)

1. **C4 Tiled U-Net** — lowest-risk fit, reuses the trained 0.37M model; the safe anchor / control.
2. **C2 Streaming TCN** — strong circuits novelty (O(state) dataflow), distillable, time-domain.
3. **C7 Conv-TasNet-style mask** — removes the U-Net skip wall (root cause) while staying time-domain; proven SE quality. The headline new candidate.

*C5 (DDR-staged) kept as the Plan-B control; C8/C9 (recompute/compressed skip) are levers to combine if a base architecture needs more headroom; C6 documents that schedule-only is not enough; C3 parked by D-2.*

> These peak-live numbers are analytical screening (Phase 1). Phase 2 trains small versions to put **quality** on the same axis and produce the quality-vs-working-set Pareto frontier before any HLS.
